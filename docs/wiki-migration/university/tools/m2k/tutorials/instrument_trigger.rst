M2k instrument trigger
======================

| The instrument trigger enables the user to capture additional data starting from the same point in time.
| Using two chained M2ks is equivalent to having a 4 channels Oscilloscope(100MSPS).
| The instrument triggering can be:

-  internal - **ADC <-> LA**
-  external - M2k daisy chain or some other instrument.

.. tip::

   The ADC trigger and logic anlyzer run on the same clock.

Using external triggers
~~~~~~~~~~~~~~~~~~~~~~~

One can simply use the **Ti** pin as an external trigger for ADC and LA. This is done by selecting the source.

++++ Show/Hide Image: \| |image3| ++++

.. important::

   By default, the **To**\ (external trigger) of an M2k will have a hold period of 2ms for each transition. High -> low or low -> high. This was done to reduce the noise caused by long wires.


Creating a daisy-chain of M2ks
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

To create a daisy-chain of M2ks, one must connect the **To** pin of the master M2k to the **Ti** of the first M2k slave. Then the **To** of the first slave to the **Ti** of the second slave and so on.

| Is there a limit on the number of devices in a chain?
| Yes. But, the limit is high enough so that it is not practical to be achieved. The limit on how many M2ks can be chained is given by the memory depth for the selected time base. Each device adds a propagation delay with it and this gives the dimension of the chain.

.. important::

   When using a daisy-chain of M2ks try to use wires as short as possible for the ground connection, to avoid the noise generated by ground coupling on long wires and the crosstalk phenomenon.


For a better understanding of the effects of long ground coupling, three setups where made.

Looking at the ADALM2000 Pinout above.

The upper M2k generates a 500KHz, 2VPP sinewave(Analog Output 1) which is connected to the Analog Input 1 of both M2ks.

Internal trigger - long GND wires (~21cm)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

All ground pins of the M2ks are connected to each other through long wires.

|image4| |image5|

The trigger is internally generated by the lower M2k. The waveform is from the same lower M2k receiving the sinewave.

External trigger - long GND wires (~21cm)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The only difference in regards to the first setup is the use of an external trigger instead of an internal one. The trigger is generated by the master M2k on its internal Oscilloscope and sent by means of **To** -> **Ti** to the second M2k, from which the waveform is displayed.

.. image:: https://wiki.analog.com/_media/university/tools/m2k/users/m2k_daisy_chain_trigger.png

As you can see there is a small noise/glitch on the sinewave. That noise is caused by ground coupling with long wires.

This type of noise is not particular for the external trigger mechanism only. This noise can be observed on the Oscilloscope when two M2ks share some Pattern Generator(PG) - Logic Analyzer(LA) signals.

External trigger - short GND wires (~3cm)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The main difference of this setup is the use of short ground connection wires. The trigger is still generated by the master M2k and passed to the slave one.

|image6| |image7| |image8|

It can be seen that for the 2VPP amplitude sampled sinewave, the noise is reduced at a level that can't be observed.

.. tip::

   As you can imagine the noise will be more visible when capturing low amplitude signals.


.. note::

   The switching noise can also be reduced by adding a 4.7K resistor in series with the connection from one M2k to the other M2k.


Trigger behavior at different sampling rates
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. important::

   When capturing/sampling a signal one should make sure that the signal does not exceed the Nyquist frequency.


.. tip::

   To simplify the following explanations the examples below will be made on square wave signals and the triggers will be on the rising edge of the data.


.. tip::

   The triggers are internally generated by axi_adc_trigger(ADC trigger) or by axi_logic_analyzer(LA).

   
   The valid signals are generated by the decimation/down-sampling filters.


.. image:: https://wiki.analog.com/_media/university/tools/m2k/users/adc_trigger_max_smp_rate.svg
   :align: center
   :width: 800px

At the maximum sampling rate, the valid is always asserted.

The trigger detect logic acts like a comparator. Every trigger event will cause a trigger pulse. Not all trigger pulses will have an effect on the system. After the DMA receives the first trigger(start sync) the other triggers will be ignored, until the DMA finishes the ongoing transaction of the requested number of samples.

.. image:: https://wiki.analog.com/_media/university/tools/m2k/users/adc_trigger_reduced_sampling_rate.svg
   :align: center
   :width: 800px

Although the above timing diagram does not reflect the filtered data(ADC path) it is useful as a conceptual example. Here you can see the effect of filtering and trigger on the samples of a fast signal.

.. image:: https://wiki.analog.com/_media/university/tools/m2k/users/instrument_trigger_ideal_env.svg
   :align: center
   :width: 800px

When using the instrument triggering the ideal system has the ADC and LA in sync, working with the same timestamp data.

.. image:: https://wiki.analog.com/_media/university/tools/m2k/users/instrument_trigger_real_env.svg
   :align: center
   :width: 800px

Because the filtering on the ADC(decimation) and LA(downsampling counter) are not the same, there can be a difference between the time of captured samples. The captured samples data can vary more than 1 sample apart between ADC and LA because of the filtering differences.

Having large and unequal time-base in the instruments will result in poorly synced instruments.

Summary
~~~~~~~

.. note::

   When capturing/sampling a signal one should make sure that the signal does not exceed the Nyquist frequency.


.. note::

   When using a daisy-chain of M2ks try to use wires as short as possible for the ground connection, to avoid the noise generated by ground coupling on long wires and the crosstalk phenomenon.


.. note::

   The switching noise can also be reduced by adding a 4.7K resistor in series with the connection from one M2k to the other M2k.


.. note::

   By default, the **To**\ (external trigger) of an M2k will have a hold period of 2ms for each transition. High -> low or low -> high. This was done to reduce the noise caused by long wires. It will have a big impact on the refresh rate.
