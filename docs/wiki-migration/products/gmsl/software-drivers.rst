Getting Started with GMSL Software Drivers
==========================================

We will walk you through the process of configuring and building a GMSL software driver.

Purpose
-------

Software drivers provide a method to communicate with the low-level hardware devices with simple and consistent implementations. They also enable a single source of truth for software distribution and allow for one code base to maintain.

The GMSL drivers enable, simple user-space configuration of the GMSL link and provide a communication channel to the image sensor for sensor driver configuration.

--------------

Software Downloads
------------------

First, begin by :doc:`Downloading and Installing the GMSL GUI </wiki-migration/products/gmsl/gui/installation>` with the latest version.


|

.. container:: group

   
   .. container:: half column

      Next, download the GMSL software driver stack.

         
         -  Create a myAnalog account with your official email ID at https://my.analog.com/en/app
         -  Once your myAnalog account is setup, send a mail to *gmsl_sw_request@analog.com* requesting the software package
         
            -  Mention the software package as: GMSL Software
            -  Also, briefly describe your GMSL use-case, your ADI contact details and your company details like name, location etc.
         
         -  Once ADI pre-approves the requested software package, a download link is sent to the email ID.
         -  Login to your `myAnalog account <https://my.analog.com/en/app>`__ and find the pre-approved software packages in ‘Resources’ > ‘Software Downloads’.
         -  You need to accept the associated licensing agreement to download the package.
