General Description
-------------------

This user guide describes the AD9084 evaluation platform and the necessary hardware and software configurations to evaluate the mixed-signal front-end RF transceiver (MxFE®).

The ACE software allows the user to quickly set up the MxFE® in multiple configurations, collectively known as Use-Cases, to play a signal out of the digital-to-analog converter (DAC) or capture analog-to-digital converter (ADC) data across the JESD204B/C link to an FPGA on the ADS10-V1EBZ. The signal data can be transferred between the ADS10-V1EBZ and a host PC for analysis, in ACE or other software. The ACE software also allows the user to generate signal vectors (signal of a finite length stored in a ASCII file, as opposed to a stream) that is transferred to the Apollo MxFE, processed by its Digital Signal Processing (DSP) engine, and reconstructed out of the DACs into a spectrum analyzer or other test equipment.

From a software perspective, the ACE software interfaces with an eRPC server hosted on the ADS10-V1EBZ. To control the AD9084 and other devices on the evaluation board, both ACE and the PyApp make calls through an eRPC server hosted on the MicroZed™. The server contains a pre-compiled version of the API, to which the host PC issues commands using a set of DLLs that are included with the software package. The MicroZed™ runs Ubuntu, a Linux distro, from a bootable SD-Card supplied with the AD9084 evaluation board.

Alternatively, users have the option to SSH to the MicroZed™ from the host PC using standard SSH clients such as PuTTY or WinSCP. This allows modifying and compiling other versions of the API, and execute the code directly on the MicroZed™ without using ACE, PyApp, or the eRPC server interface. Together with the Apollo MxFE API package included are Example c-files that may serve as a reference to the developer to quickly configure the evaluation platform through the API, port over any custom API code from a prototype platform and compare the results. While the API specifically control the Apollo MxFE installed on the Apollo MxFE evaluation board, the code included in the Examples folder also controls the ADS10-V1EBZ and any peripheral devices on the evaluation board, such as on-board clocking or other on-board devices that may be paired with the AD9084, to serve as complete system reference design.

Signal vectors and captured ADC data is transferred between the ADS10-V1EBZ and the host PC across an ethernet link. Certain features and DSP functionality may be initially available only in the API until support for these features is added to the Apollo MxFE Python Application (PyApp) and the ACE software.

AD9084-FMCB-EBZ
~~~~~~~~~~~~~~~

The AD9084-FMCA-EBZ has RF baluns in the ADC and DAC RF signal paths performing single-ended to differential conversion. AD9084-FMCB-EBZ has an ADL6332 RF amplifier on each of the 4 ADC input paths and an ADL6331 RF amplifier on 4 of the DAC signal paths.

`image4.img <https://wiki.analog.com/_media/image4.img>`__

*Figure 4 AD9084-FMCB-EBZ Amplifiers and Filters*

The AD9084-FMCB-EBZ supports L/S/C band on Side A and up to X band on side B for both DAC and ADC channels. Additionally, there are two DAC channels with balun that can transmit full Apollo bandwidth. The integrated 50-ohm differential lowpass filters are capable of rejecting harmonics and DAC images. Tx and Rx VGAs (ADL6331 and ADL6332) offer variable gain with 24dB dynamic range.

Setting Up the MicroZed™
------------------------

The MicroZed™ is an embedded computer board, installed as a mezzanine board on the ADS10-V1EBZ, that can boot (initialize) an operating system from bootable media such as an sd-card. One of the sd-cards supplied with the FMCA/FMCB evaluation board contains a version of the Linux distro Ubuntu, which stores a pre-compiled version of the API with which ACE and other software from Analog Devices can interface using a set of Dynamic Link Libraries (DLLs) that communicate across an eRPC server hosted on the microZed.

Jumpers on the MicroZed™ must be configured to indicate to the MicroZed™ which media it should boot from, namely the sd-card.

Once the MicroZed™ finished booting, the host PC can establish an ethernet link to the MicroZed™, access its memory and folder structure at the root-level, and manipulate the various APIs to evaluate the AD9084 and any of its companion devices that may be located on the AD9084-FMCx-EBZ board.

Limitations
"""""""""""

**\*EXAMPLE OF CONFIGURATION/LIMITATION. PLEASE WRITE THIS KIND OF INFORMATION IF IT APPLIES TO THE PROJECT**\ \*

The design has one JESD receive chain with 4 lanes at rate of 13Gbps. The JESD receive chain consists of a physical layer represented by an XCVR module, a link layer represented by an RX JESD LINK module and transport layer represented by a RX JESD TPL module. The link operates in Subclass 1.

The link is set for full bandwidth mode and operate with the following parameters:

Deframer paramaters: L=4, M=2, F=1, S=1, N’=16

| SYSREF - 5.078125 MHZ
| REFCLK – 325MHz (Lane Rate/40)
| DEVICECLK -325 MHz
| ADCCLK – 1300MHz
| JESD204B Lane Rate – 13Gbps

The transport layer component presents on its output 128 bits at once on every clock cycle, representing 4 samples per converter. The two receive chains are merged together and transferred to the DDR with a single DMA.
