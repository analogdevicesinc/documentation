AD5706R IIO DAC Linux Driver
============================

Supported Devices
-----------------

This driver supports the

-  :adi:`AD5706R` To be updated on release

Evaluation Boards
-----------------

-  :adi:`EVAL-AD5706RARDZ` To be updated on release

Description
-----------

This is a Linux industrial I/O (:doc:`IIO </wiki-migration/software/linux/docs/iio/iio>`) subsystem driver, targeting multi-channel serial interface DACs. The industrial I/O subsystem provides a unified framework for drivers for many different types of converters and sensors using a number of different physical interfaces (i2c, spi, etc). See :doc:`IIO </wiki-migration/software/linux/docs/iio/iio>` for more information.

Devicetree
----------

Required devicetree properties for the AD5706R:

-  compatible: Must be "adi,ad5706r"
-  reg: SPI chip select number for the device
-  spi-max-frequency: Max SPI frequency to use (< 5000000)
-  pwms: The built in IP of a PWM Generator.

Leave the FPGA AXI untouched, except 'label' which is equal to device property 'pwm-names'.

Optional properties:

-  dac-resetb-gpios: GPIO spec for the RESET pin. If specified, it will be asserted during driver probe.
-  dac-mdX_a0-gpios: GPIO spec for Device 'X' Address bit 0 pin. Device number can be from 0-3.
-  dac-mdX_a1-gpios: GPIO spec for Device 'X' Address bit 1 pin. Device number can be from 0-3.

::

   &fpga_axi {

       axi_pwm_gen: pwm@0x44a00000 {
           compatible = "adi,axi-pwmgen";
           reg = <0x44a00000 0x1000>;
           label = "ad5706r_ldacb";
           #pwm-cells = <2>;
           clocks = <&clkc 15>;
       };
   };

   &adc {
       
           #io-channel-cells = <1>;    
           xlnx,channels {
                   #address-cells = <1>;
                   #size-cells = <0>;
                   channel@2 {
                           reg = <2>;
                           label = "adc_channel2";
                   };
           };
   };
           
   &spi0 {
       status = "okay";

           io-channels = <&adc 9>;
           io-channel-names = "mux_out_raw2";
                   
           
       ad5706r: dac@0 {
           compatible = "adi,ad5706r";
           reg = <0>;
           spi-max-frequency = <5000000>;
                   
           pwm-names = "ad5706r_ldacb";
           pwms = <&axi_pwm_gen 0 0>;
                   
                   //address is 54+hdl GPIO Number
                   dac-resetb-gpios = <&gpio0 86 GPIO_ACTIVE_LOW>;
                   dac-shdn-gpios   = <&gpio0 87 GPIO_ACTIVE_HIGH>;
                   dac-md0_a0-gpios = <&gpio0 88 GPIO_ACTIVE_HIGH>;
                   dac-md0_a1-gpios = <&gpio0 89 GPIO_ACTIVE_HIGH>;
                   dac-md1_a0-gpios = <&gpio0 90 GPIO_ACTIVE_HIGH>;
                   dac-md1_a1-gpios = <&gpio0 91 GPIO_ACTIVE_HIGH>;
                   dac-md2_a0-gpios = <&gpio0 92 GPIO_ACTIVE_HIGH>;
                   dac-md2_a1-gpios = <&gpio0 93 GPIO_ACTIVE_HIGH>;
                   dac-md3_a0-gpios = <&gpio0 94 GPIO_ACTIVE_HIGH>;
                   dac-md3_a1-gpios = <&gpio0 95 GPIO_ACTIVE_HIGH>;
               
       };
   };

                   ...
                   <*> Analog Devices AD5706R DAC Driver
                   ...
               ...
           ...

Driver testing
==============

Each and every IIO device, typically a hardware chip, has a device folder under /sys/bus/iio/devices/iio:deviceX. Where X is the IIO index of the device. Under every of these directory folders reside a set of files, depending on the characteristics and features of the hardware device in question. These files are consistently generalized and documented in the IIO ABI documentation. In order to determine which IIO deviceX corresponds to which hardware device, the user can read the name file /sys/bus/iio/devices/iio:deviceX/name. In case the sequence in which the iio device drivers are loaded/registered is constant, the numbering is constant and may be known in advance.

Set automatic register address increment/decrement via 'addr_ascension'
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

| **Description:**
| /sys/bus/iio/devices/iio:deviceX/addr_ascension
| During multiple register read or write, this sets the direction whether the next address is an increment or decrement of the previous.

Available Values: decrement, increment

Default: decrement

.. container:: box bggreen

   
   .. note::

      This specifies any shell prompt running on the target

      root@analog:/sys/bus/iio/devices/iio:device1# cat addr_ascension
      increment
