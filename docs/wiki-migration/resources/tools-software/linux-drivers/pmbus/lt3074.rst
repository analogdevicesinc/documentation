LT3074 Ultra-fast Linear Regulator with PMBus
=============================================

Supported Devices
-----------------

-  :adi:`LT3074`

Description
-----------

This client driver is dependent to the core PMBus driver. The `PMBus driver <https://www.kernel.org/doc/Documentation/hwmon/pmbus>`__ is responsible for exposing a set of sysfs file required for the PMBus interface. Additionally, the LT3074 driver also adds debugfs entries to allow access to other PMBus commands and manufacturer-specific registers.

Example I2C device initialization
=================================

Currently, the LT3074 driver can only be instantiated via devicetree.

Required devicetree properties:

-  **compatible**: Must be always one of these:

   -  adi,lt3074

-  **reg**: I2C device base address.

::

   &i2c1 {
           #address-cells = <1>;
           #size-cells = <0>;

           lt3074@1d {
                   compatible = "adi,lt3074";
                   reg = <0x1d>;
           };
   };

Enabling Linux driver support
=============================

Configure kernel with “make menuconfig” (alternatively use “make xconfig” or “make qconfig”).

.. hint::

   The LT3074 driver is a client driver of the PMBus core driver which depends on **CONFIG_I2C**.

HWMON device files
------------------

All sysfs files of a PMBus driver can be found under /sys/class/hwmon/hwmonX, where X is the hwmon device index. These files contain voltage, temperature and current data of the device. To determine which hwmon device corresponds to which hardware chip, the user can read the name file under /sys/class/hwmon/hwmonX/name. Meanwhile, if debugfs is enabled, all debugfs files are found under /sys/kernel/debug/pmbus/hwmonX. Status and manufacturer-specific registers can be accessed here.

For more information on the files format, take a look to `sysfs-interface <https://www.kernel.org/doc/Documentation/hwmon/sysfs-interface>`__.

The PMBus core driver exposes sysfs files with the format {channel}\_{attribute}. Below is a mapping of the sysfs file name prefixes to their monitored values:

-  **in1\_**: Input voltage
-  **in2\_**: Bias voltage
-  **in3\_**: Output voltage
-  **curr1\_**: Output current

Below is a mapping of the sysfs file name suffixes to their corresponding attributes:

-  **\_label**: Label name
-  **\_input**: Voltage/current data
-  **\_max**: Overvalue warning limit
-  **\_min**: Undervalue warning limit
-  **\_max_alarm**: Overvalue warning status
-  **\_min_alarm**: Undervalue warning status
-  **\_crit**: Overcurrent fault limit
-  **\_crit_alarm**: Overcurrent fault status
-  **\_lcrit**: Undercurrent warning limit
-  **\_lcrit_alarm**: Undercurrent warning status

Set margin percentages
~~~~~~~~~~~~~~~~~~~~~~

Setting the margin percentage is automatically rounded down to the nearest defined setting as stated in the datasheet.

.. container:: box bggreen

   
   .. note::

      This specifies any shell prompt running on the target

   
   ::
   
      root:/sys/kernel/debug/pmbus/hwmon2/lt3074> **echo 27 > margin_high**
      root:/sys/kernel/debug/pmbus/hwmon2/lt3074> **cat margin_high**
      25
