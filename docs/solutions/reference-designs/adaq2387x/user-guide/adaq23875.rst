ADAQ23875 User Guide
===============================================================================

Evaluating Using ZedBoard

Features
-------------------------------------------------------------------------------

- Evaluation board for the :adi:`ADAQ23875` (16-bit, 15 MSPS) µModule Data
  Acquisition Solutions
- Integrated fully differential ADC driver with signal scaling
- On-board reference buffer with VCMO generation
- PC software for control and data analysis of time and frequency domain

Evaluation Requirements
-------------------------------------------------------------------------------

Hardware
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- :adi:`EVAL-ADAQ23875FMCZ <EVAL-ADAQ23875>` evaluation board
- `ZedBoard (AES-Z7EV-7Z020-G)
  <https://www.xilinx.com/products/boards-and-kits/1-8dyf-11.html>`__
- 12V power supply
- Host PC running Windows 10 or higher
- SD Card (16GB or larger)
- LAN Cable
- SMA Cable
- XLR to SMA adapter cable
- Audio Analyzer (Audio Precision APX525) or other input source

Software
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- :external+kuiper:doc:`Analog Devices Kuiper Linux <index>` image

General Description
-------------------------------------------------------------------------------

The :adi:`EVAL-ADAQ23875FMCZ <EVAL-ADAQ23875>` evaluation board enables
simplified evaluation of the :adi:`ADAQ23875` 15 MSPS, 16-bit, High Speed,
Precision µModule Data Acquisition Solutions. The µModule integrates passive
components with superior matching and drift characteristics to minimize
temperature-dependent error sources.

Getting Started
-------------------------------------------------------------------------------

Setting Up the Evaluation Board
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The evaluation board consists of:

- **U1**: :adi:`ADAQ23875` µModule
- **U5**: :adi:`LTC6655` 4.096V reference, or **U3**: :adi:`ADR4520` 2.048V
  reference
- **U6**: :adi:`LTM8049` power µModule
- **U4**: :adi:`ADP7118`
- **U7**: :adi:`ADP7183`
- **U8**: :adi:`LT3023`
- **U13**: 800 MHz clock distribution IC
- Optional amplifiers: :adi:`ADA4899-1` (A2, A3)

Power Supplies
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. figure:: images/table1.png
   :align: center

   Table 1: On-Board Power Supplies

The :adi:`EVAL-ADAQ23875FMCZ <EVAL-ADAQ23875>` can be powered from an
external 3.3V supply via the JP9 solder link.

The positive supply rails generated are:

- 7V (+VS)
- 5V (VDD)
- 2.5V (VIO)

The negative supply rail generated is:

- −2.0V (−VS)

Components used: :adi:`LTM8049` (U6), :adi:`ADP7118` (U4), :adi:`LT3023`
(U8), :adi:`ADP7183` (U7).

Each supply rail has the necessary decoupling capacitors placed as close as
possible to the device. A single ground plane is used to minimize the effect
of high-frequency noise interference.

Analog Inputs
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Subminiature Version A (SMA) connectors (VIN+ and VIN−) provide analog inputs
from a low-noise, audio precision signal source (SYS-2700 or SYS-x555 series).

Three options are available for feeding the input signals:

#. Feed directly to the :adi:`ADG774` analog switch
#. Feed through the optional :adi:`ADA4899-1` amplifiers (A2, A3)
#. Feed directly to the :adi:`ADAQ23875`

The optional :adi:`ADA4899-1` amplifiers (A2, A3) can be configured in
unity-gain, driving the :adi:`ADAQ23875`. The default configuration allows the
input signal via VIN+ and VIN− to feed directly to the :adi:`ADAQ23875` by
bypassing U2, A2, and A3.

The default board configuration provides 4.096V on the REFBUF pin and a
buffered 2.048V (midscale) at the VCMO pin.

.. note::

   For low input frequency testing below 100 kHz, use a low-noise audio
   precision signal source (SYS-2700 series) with outputs set to balanced
   floating.

Loading the Image onto the SD Card
-------------------------------------------------------------------------------

To properly evaluate the :adi:`EVAL-ADAQ23875FMCZ <EVAL-ADAQ23875>`, the ADI
Kuiper Linux must be properly flashed on the SD card. Complete instructions are
available at :external+kuiper:doc:`Kuiper Linux <index>`.

Follow the directions for preparing the image specifically for the
EVAL-ADAQ23875FMCZ.

SD Card Preparation
-------------------------------------------------------------------------------

#. **Download the ADI Kuiper Image**

   Follow the instructions at :external+kuiper:doc:`Kuiper Linux <index>`.
   The Kuiper Linux image is an open-source embedded Linux operating system
   based on Raspberry Pi OS, incorporating Linux device drivers for ADI
   products.

   .. note::

      Use the image for the **CN0577** project.

   The ZedBoard can be accessed from a remote PC via LAN cable or Wi-Fi.

#. **Format and Flash the SD Card on Windows hosts**

   Follow the :dokuwiki:`Windows hosts instructions
   <resources/tools-software/linux-software/zynq_images/windows_hosts>`.

#. **Format and Flash the SD Card on Linux hosts**

   Follow the :dokuwiki:`Linux hosts instructions
   <resources/tools-software/linux-software/zynq_images/linux_hosts>`.

After flashing, ensure the SD card is configured for the correct FPGA project
per the :external+kuiper:doc:`Kuiper Linux <index>` documentation.

Copy the following key files to the root folder of the SD card:

#. ``BOOT.BIN``
#. ``devicetree.dtb``
#. ``uImage`` (from the ``zynq-common`` folder)

Hardware Connection
-------------------------------------------------------------------------------

.. figure:: images/hardware_connection_adaq23875.png
   :align: center

   Figure 2: Signal Chain Connection

Setup steps (using the Audio Precision Audio Analyzer as input source):

#. Download and install the IIO Oscilloscope application on the host PC.
#. Flash the Analog Devices Kuiper Linux image on the SD card.
#. Properly configure the SD card for the EVAL-ADAQ23875FMCZ.
#. Insert the SD card into the ZedBoard.
#. Attach the EVAL-ADAQ23875FMCZ to the ZedBoard through the FMC pin connector.
#. Power up the ZedBoard using the 12V cable.
#. Connect the EVAL-ADAQ23875FMCZ to the Audio Precision Audio Analyzer using
   the XLR to SMA adapter cable.
#. Connect the Audio Precision Audio Analyzer USB cable to the PC.

After completing these steps, the IIO Oscilloscope software should run smoothly.

Navigating the IIO Oscilloscope
-------------------------------------------------------------------------------

Main Window
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. figure:: images/main_window.png
   :align: center

   Figure 3: Main Window

The main window displays a configuration panel (plugin) for each device
recognized by the system. Additional plugins are loaded for device debugging
and monitoring (Debug Tab/Plugin and DMM Tab/Plugin).

Each plugin can be detached by clicking the button on the right side of the
plugin name. Close a detached window to re-attach it to the main window.

Debug Tab/Plugin
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. figure:: images/debug_tab.png
   :align: center

   Figure 4: Debug Tab/Plugin

The Debug plugin is a tool for device debugging. Some features may not work
unless the oscilloscope application is started as root.

#. **Device Selection:** Sets the active device.
#. **IIO Device Attributes:** Allows read/write operations for device
   attributes.
#. **Register:** Provides low-level access to device registers.

   a. *Enable AutoRead:* Allows automatic register reading as the register
      address changes.
   b. *Address:* Register address field.
   c. *Value:* Register value at the given address.

DMM Tab/Plugin
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. figure:: images/dmm_tab.png
   :align: center

   Figure 5: DMM Tab/Plugin

The DMM (Digital Multimeter) plugin displays device-specific data continuously
once the start button is activated.

#. **Device tab:** Lists all available devices.
#. **Active channels tab:** Lists the channels of enabled devices. The
   "All Channels" button enables all channels simultaneously.
#. **Right side tab:** Displays data readings of the enabled channels.

Capture Window
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**Menu (along top):**

- **File**

  - *Save As:* Saves data to a file.
  - *Quit:* Closes the capture/plot window.

- **Edit**

  - *Plot Title:* Changes the window name.

- **View**

  - *Show Settings:* Shows/hides the settings panel on the left, allowing the
    plot to fill the entire window.
  - *Full Screen:* Makes the window full screen.

**Plot options/icons (along top):**

- *Capture/Stop:* Start or stop data capture.
- *Zoom In:* Zoom in on the plot region.
- *Zoom Out:* Zoom out from the plot region.
- *AutoZoom:* Automatic zoom to fit the screen.
- *Save As:* Save data to a file.
- *FullScreen:* Enter/leave fullscreen mode.
- *Auto scale:* Automatically re-scale the visible area to fit the entire plot.
- *Show grid:* Show or hide the grid in the plot window.
- *Y Max:* Adjust the upper limit of the vertical axis (when Auto scale is
  disabled).
- *Y Min:* Adjust the lower limit of the vertical axis (when Auto scale is
  disabled).
- *New Plot:* Create a new plot of the same type.

Activating Markers
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Markers are used for plot data measurement in the frequency domain or during
cross correlation. Right-click on the plot and select a marker type to enable.

Available marker types:

- FFT domain (1 channel enabled): Peak, Fixed, Single Tone
- FFT domain (2 channels enabled): Peak, Fixed, Single Tone, Image
- Constellation: Peak Marker

Five markers are displayed by default. Add more via *Add Marker* or remove via
*Remove Marker*.

Board Layout Guidelines
-------------------------------------------------------------------------------

PCB layout is critical for preserving signal integrity and achieving the
expected performance from the :adi:`ADAQ23875`.

- Use a multilayer board with an internal, clean ground plane in the first
  layer beneath the :adi:`ADAQ23875`.
- Apply careful placement of individual components and routing of signals.
- Route input and output signals symmetrically.
- Solder the ground pins of the :adi:`ADAQ23875` directly to the ground plane
  using multiple vias.
- **Remove ground and power planes** under the analog input/output and digital
  input/output pins (including F1 and F2) of the :adi:`ADAQ23875` to avoid
  undesired parasitic capacitance. Undesired parasitic capacitance impacts
  distortion and linearity performance.

**Signal routing:**

The pinout of the :adi:`ADAQ23875` eases layout by placing analog signals on
the left side and digital signals on the right side.

- Separate sensitive analog and digital sections on the PCB.
- Keep power supply circuitry away from the analog signal path.
- Fast switching signals (CNV±, CLK±) and DA±, DB± digital outputs must not
  run near or cross analog signal paths.

**Decoupling:**

- Use good-quality ceramic bypass capacitors of at least 2.2 µF (0402, X5R)
  at the output of each LDO regulator generating the µModule supply rails
  (VDD, VIO, VS+, VS−) to GND.
- This minimizes electromagnetic interference (EMI) susceptibility and reduces
  glitch effects on the power supply lines.
- All other required bypass capacitors are laid out within the :adi:`ADAQ23875`.
- External decoupling capacitors on the REFIN, VDD, and VIO pins near the
  µModule can be removed with no significant impact on performance.

Mechanical Stress
-------------------------------------------------------------------------------

Mechanical stress from mounting the device to the board can cause subtle
changes to SNR and the internal voltage reference.

The recommended soldering method is IR reflow or convection soldering with a
controlled temperature profile. Hand soldering with a heat gun or soldering
iron is **not** recommended.
