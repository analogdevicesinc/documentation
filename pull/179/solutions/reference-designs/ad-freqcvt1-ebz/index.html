AD-FREQCVT1-EBZ

The AD-FREQCVT1-EBZ is a frequency up/down converter add-on board for the FMCOMMS3/4/5 boards which contain the AD9361. The purpose of the AD-FREQCVT1-EBZ is to extend the AD9361 operating range down to 1 MHz through frequency up and down conversion.

Introduction

The AD-FREQCVT1-EBZ module is comprised of the AD8342 mixer, the ADL5530 LNA, two ADF4351 synthesizers, and power management components. It is controlled via the PMOD port on the FPGA carrier board.

For additional information or to purchase the AD-FREQCVT1-EBZ, visit the AD-FREQCVT1-EBZ product page.

Features

  • Add-on to the FMCOMMS3/4/5 family

  • Powered from the FMCOMMS board

  • Provides one channel down-conversion transmit and one channel up-conversion receive

  • Extends AD9361 frequency range down to 1 MHz

Applications

  • Electronic test and measurement equipment

  • General-purpose software radios

  • Radar systems

  • Ultra wideband satellite receivers

  • Signals intelligence (SIGINT)

  • Point-to-point communication systems

  • Automated test equipment

Functional Overview

The AD-FREQCVT1-EBZ extends the frequency of operation for the FMCOMMS3/4/5 boards:

  • Input signals in the frequency range of 1–100 MHz are up-converted to an output signal centered around 340 MHz to the FMCOMMS board.

  • An output signal in the range of 1–100 MHz is generated by down-converting a 370 MHz signal from the FMCOMMS board.

  • On-board programmable phase-locked loop frequency synthesizers generate the needed local oscillator signals used by the up- and down-converting mixers.

Local Oscillator

The local oscillator used on this board is the ADF4351, which is software programmable. Two ADF4351 instances provide the TX and RX LO signals. The board provides a 25 MHz crystal as the reference for the ADF4351.

SPI Interface

The SPI signals are directly passed through to the FMC connector. A 12-pin ribbon cable connects the PMOD header on the FPGA carrier board to the AD-FREQCVT1-EBZ digital control connector (P2), implementing the SPI protocol and general-purpose I/Os used to control the two on-board ADF4351 synthesizers.

Power Supply

The board receives all power from the FMCOMMS board through the power supply cable provided. Key power management components:

Quick Start Guide

The Quick Start Guide provides step-by-step instructions detailing how to connect and set up the AD-FREQCVT1-EBZ board. The ZC706 and AD-FMCOMMS3-EBZ platforms are used to demonstrate the appropriate board and cable connections.

Prerequisites

  • ZC706 carrier board and power supply

  • AD-FMCOMMS3-EBZ (or FMCOMMS4/5) board

  • AD-FREQCVT1-EBZ board

  • 12-pin ribbon cable (included)

  • 3-pin power cable (included)

  • 2 male-to-male SMA cables

  • SMA loopback cable (for testing)

  • SD card with Kuiper Linux image

  • HDMI display, USB keyboard and mouse

Connecting to FMCOMMS3/4 and Cable Orientation

The AD-FREQCVT1-EBZ has four SMA connectors (TX_IN, RX_OUT, TX_OUT, RX_IN). Two male-to-male SMA connectors are required to connect the AD-FREQCVT1-EBZ board to the FMCOMMS3 platform.

To use the second channel of the FMCOMMS3, line up FREQCVT1 connectors RX_OUT and TX_IN with FMCOMMS3 connectors TX2A and RX2A.

To use the first channel of the FMCOMMS3, line up FREQCVT1 connectors RX_OUT and TX_IN with FMCOMMS3 connectors RX1A and TX1A. This requires flipping the board upside down.

The ZC706 board provides digital controls to the AD-FREQCVT1-EBZ board through the 12-pin ribbon cable. Be careful to properly plug in both ends of the cable according to the image above.

Power Cable Connection

Power is brought to the AD-FREQCVT1-EBZ board through a 3-pin cable. The connection is made at P1 (pin 1 = 3.3 V, pin 2 = GND, pin 3 = 12 V), which sits directly between TX_IN and RX_OUT. The power originates from the FMCOMMS3 board, connector P301.

Important

It is critical to properly plug in the power cable on both ends. Connector P301 on the FMCOMMS3 board is not populated and requires a 3-pin male header. The cable must be carefully plugged into P301 to ensure the correct voltage is applied. The connector on the AD-FREQCVT1-EBZ board (P1) is keyed.

Software

Theory of Operation

The AD-FREQCVT1-EBZ extends the frequency of operation for the FMCOMMS3/4. Two on-board programmable phase-locked loop frequency synthesizers (ADF4351) generate the needed local oscillator signals used by the up- and down-converting mixers. Input signals in the frequency range of 1–100 MHz are up-converted to an output signal centered around 340 MHz (RX SAW BPF) to the FMCOMMS board. An output signal in the range of 1–100 MHz is generated by down-converting a 370 MHz (TX SAW BPF) signal from the FMCOMMS board.

The Analog Devices board support packages for the FMCOMMS3/4 design, on the various supported FPGA carriers, include an additional SPI interface routed to the PMOD board header.

Two ADF4351 driver instances control the external synthesizers. The IIO Oscilloscope FMCOMMS2/3/4 plugin implements the external and internal LO synthesizer tuning logic. When the external synthesizers are detected, an additional checkbox in the plugin is displayed, allowing the user to enable or disable the tuning logic. Once enabled, the RX/TX LO spin button switches its range from 70 MHz–6 GHz to 1 MHz–100 MHz.

Environment Variables

Two environment variables allow the user to control the sign in the mixing equation as well as the window size (span) where the signal is centered around the SAW band-pass filter center frequency.

Production Testing

The production testing process verifies each AD-FREQCVT1-EBZ board for gross errors. The test first checks the noise floor with no input signal to ensure it is within threshold, then inputs test tones and verifies the response is within expected bounds.

Creating a Test SD Card

  1. Write the latest Kuiper Linux image to an SD card.

  2. Copy the BOOT.BIN and devicetree.dtb files from the zynq-zc706-adv7511-ad9361-fmcomms2-3 folder into the boot partition.

  3. Modify the card to run tests automatically on boot. Test scripts from the linux_image_ADI-scripts repository automate initializing the IIO Oscilloscope with the correct profile and environment. Update the desktop autostart launcher to execute the test script instead of IIO Oscilloscope.

Required Hardware

  • ZC706 carrier board and power supply

  • HDMI monitor, USB keyboard and mouse

  • SD test card (prepared above)

  • AD-FMCOMMS3-EBZ board

  • AD-FREQCVT1-EBZ board

  • 12-pin ribbon cable

  • 3-pin power cable

  • SMA loopback cable

Test Process

  1. Ensure the ZC706 is set up with the testing SD card, HDMI monitor, and keyboard/mouse connected and powered on.

  2. Mount the AD-FREQCVT1-EBZ board on the TX2A/RX2A jacks of the AD-FMCOMMS3-EBZ board. Connect the power cable and signal cables as shown below.

  3. Connect the RX_IN and TX_OUT jacks on the AD-FREQCVT1-EBZ board with the SMA loopback cable.

  4. Power on the ZC706 board. The system boots into the ADI Linux environment and begins tests automatically.

  5. When tests pass, a confirmation window is displayed. If tests fail, an error dialog appears instead. Confirm the result to halt Linux, then power off the carrier board.

  6. Remove the AD-FREQCVT1-EBZ board and repeat the process for the next unit.
