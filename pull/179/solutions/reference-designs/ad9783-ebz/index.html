AD9783-EBZ User Guide

Introduction

The AD9783 includes pin-compatible, high dynamic range, dual digital-to-analog converters (DACs) with 16-bit resolution and a sample rate of up to 500 MSPS. The device includes specific features for direct conversion transmit applications, including gain and offset compensation, interfacing seamlessly with analog quadrature modulators.

The reference design consists of a DDS module and an LVDS interface. The AD9783-EBZ evaluation board connects to the carrier through a DAC-FMC interposer board.

This evaluation guide describes both the hardware and software setup needed to characterize the AD9780/AD9781/AD9783 12-/14-/16-bit, 500 MSPS high-speed digital-to-analog converters using the AD9783-DPG2-EBZ evaluation board with the ADS7-V2 or SDP-H1 controller boards.

Supported Devices

  • AD9780

  • AD9781

  • AD9783

Supported Carriers

  • ZCU102 (via DAC-FMC interposer board)

Evaluation Setup

Equipment Needed

  • AD9783-DPG2-EBZ evaluation board

  • SDP-H1 (EVAL-SDP-CH1Z) board or ADS7-V2 (ADS7-V2EBZ) board

  • AD-DAC-FMC-ADP high-speed DAC evaluation board to FMC adapter board

  • 5 Vdc, 1 A power supply for the evaluation board

  • High-frequency continuous wave generator (clock source)

  • Signal/spectrum analyzer

  • 2 x SMA cables

  • 2 x banana plug cables

  • PC with ACE and DPG Lite software

Additional items for SDP-H1 evaluation kit:

  • 12 Vdc, 1 A wall adapter

  • 2 x USB-A to USB-Mini cables

Additional items for ADS7-V2 evaluation kit:

  • 12 Vdc power supply with power cord

  • USB-A to USB-B cable

Software Needed

  • Analysis | Control | Evaluation (ACE)

  • DPG Lite (recommended; installed with ACE)

FPGA IP Core

The axi_ad9783 IP core provides the following features:

  • AXI Memory-Mapped to Streaming control/status interface

  • PRBS monitoring

  • Internal DDS

  • BIST testing

  • Supports only Xilinx devices

The core architecture contains an LVDS interface module, a transmit module with per-channel DAC processing (DDS, pattern, PRBS generators, and channel register map), a DAC common register map, and AXI control/status modules.

The parallel data interface consists of up to 18 differential signals: dac_clk_out_*, dac_clk_in_*, and up to 16 data lines (dac_data_out_*). DCO is the output clock generated by the AD9783 used to clock out data. DCI provides timing information about the parallel data and signals the I/Q status. Data processed on the rising edge is output on the I DAC, and data on the falling edge is output on the Q DAC.

Quick Start Guide (ACE/DPG Lite)

  1. Attach the AD9783-DPG2-EBZ to the controller board’s (SDP-H1 or ADS7-V2) FMC connector using the AD-DAC-FMC-ADP adapter board.

  2. Connect the controller board (SDP-H1 or ADS7-V2) to the PC via USB and to a 12 V power supply.

  3. Connect the evaluation board to the PC via USB and to a 5 Vdc, 1 A power supply on P5 and P6. Refer to the setup figures above.

  4. Connect a continuous wave generator for clock input to SMA J1 and the DAC output from SMA J5 (IOUT1P) or SMA J9 (IOUT2P) to a signal/spectrum analyzer.

  5. Set the clock input to 200 MHz, 3 dBm.

  6. Open ACE. The board will be automatically recognized by the software. Click the plugin. If the board is not auto-detected, install the AD9783 ACE plugin manually.

  7. On the ACE Board View, double-click the AD9783 block to access the Chip View.

  8. The default register values are displayed. Values can be changed on the Chip View GUI or via the Memory Map for registers not available in the GUI. Once the desired values are set, click Apply Changes and then Read All in the upper-right window.

  9. Start DPG Lite. The controller board (SDP-H1 or ADS7-V2), the device part number, and clock frequency should be displayed.

  10. In DPG Lite, from the Add Generator Waveforms pulldown menu, select Single Tone and apply the following settings:

    • Data Rate: 200 MHz

    • Desired Frequency: 16 MHz

    • DAC Resolution: 16 for AD9783, 14 for AD9781, 12 for AD9780

    • Uncheck Unsigned Data

    • Check Generate Complex Data (I & Q)

  11. Select the I/Q tone from the I/Q Data Vector pulldown menu.

  12. Press the download arrow and then the play button. A spectrum similar to the figure below should appear on the signal/spectrum analyzer.

Quick Start Guide (FPGA/ZCU102)

The AD9783-EBZ can also be evaluated using the ZCU102 FPGA carrier board running Linux.

Equipment Needed

  • ZCU102

  • AD9783-DPG2-EBZ evaluation board with 5 Vdc, 1 A power supply

  • AD-DAC-FMC-ADP interposer board

  • Signal/clock generator (2.5 GHz)

  • Spectrum analyzer

  • 2 x SMA cables

  • 1 x USB Mini-B cable (for evaluation board)

  • 1 x USB Micro-B cable (for UART)

  • SD card with ADI Kuiper Linux

Setup

  1. Hardware adjustments: Desolder XJP1, XJP3, XJP4, XJP5 from the AD9783-DPG2-EBZ to route SPI communication through a cable to the ZCU102 PMOD0 connector. Verify that there is no current path through the desoldered jumpers.

  2. Attach the AD9783-DPG2-EBZ to the ZCU102 FMC 0 connector using the AD-DAC-FMC-ADP adapter board.

  3. Connect a 5-wire cable from the AD9783-EBZ XP3 header to the ZCU102 PMOD0 connector for SPI communication.

  4. Prepare the SD card with the latest ADI Kuiper Linux image. From the zynqmp-common folder, copy the Image file to the boot partition root. From the zynqmp-zcu102-rev10-ad9783 folder, copy system.dtb and BOOT.BIN to the boot partition root.

  5. Insert the SD card into the ZCU102.

  6. Connect the clock source to SMA J1 on the evaluation board and the DAC output from SMA J5 (IOUT1P) or SMA J9 (IOUT2P) to a spectrum analyzer.

  7. Connect the ZCU102 UART port via USB Micro-B to the PC.

  8. Set the clock input to 500 MHz, +3 dBm.

  9. Connect the evaluation board to the 5 Vdc, 1 A power supply on P5 and P6.

  10. Power on the evaluation board, clock generator, and ZCU102.

Troubleshooting

If unexpected results occur, check the following:

  • Verify the voltage rails of the evaluation board: P5 and P6 should be 5 V; TP3, TP5, and TP7 should be 3.3 V; TP9 and TP11 should be 1.8 V.

  • Check that all three blue LEDs on the AD-DAC-FMC-ADP board are lit. Reconnect the board to the FMC connector of the controller board if the LEDs are not lit.

  • Verify that the controller board (SDP-H1 or ADS7-V2) is being supplied properly by the 12 Vdc adapter. Some LEDs on the controller board should be lit.

  • Perform a chip reset by pressing S2 on the AD9783 evaluation board.

  • Power cycle both the controller board and the AD9783 evaluation board.

  • Probe C18 and C19 to verify that a clock signal is being sent to the DAC.

  • Check that XD1 is lit on the evaluation board. Reconnect or replace the USB cable if it is not lit.

  • Disconnect and reconnect the controller board and AD9783 evaluation board. Reopen ACE and DPG Lite software.
